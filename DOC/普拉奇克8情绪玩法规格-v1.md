# 普拉奇克8情绪玩法规格（v1）

## 1. 目标

将现有情绪玩法改为：

1. 从 polyu-storyworld 角色库随机抽取角色（YAML）。
2. 调用大模型分析角色情绪，输出 8 基础情绪权重。
3. 玩家每轮猜测 3 种情绪（不设置猜错惩罚，不阻断流程）。
4. 根据玩家猜测的 3 种情绪生成调酒目标。
5. 若命中角色权重前 3 的情绪越多，则获得更高小费与信任值。

---

## 2. 情绪集合与标识

固定 8 种基础情绪（内部 ID）：

- joy（Joy / 喜悦）
- trust（Trust / 信任）
- fear（Fear / 恐惧）
- surprise（Surprise / 惊讶）
- sadness（Sadness / 悲伤）
- disgust（Disgust / 厌恶）
- anger（Anger / 愤怒）
- anticipation（Anticipation / 期待）

约束：

1. 游戏内所有情绪推理、校验、存档均以以上 8 个 ID 为唯一合法值。
2. 模型返回非合法值时，必须过滤并走降级策略。

---

## 3. 角色抽取规则

数据源：polyu-storyworld 子模块中的 characters 目录。

抽取规则：

1. 仅纳入扩展名 .yaml/.yml 的文件。
2. 随机等概率抽取一个角色文件。
3. 若文件解析失败，则重抽一次；连续失败超过上限时使用本地降级角色模板。

建议额外字段抽取（用于情绪分析）：

- name
- profile / description
- background / biography
- personality / traits
- dialogue_style（若存在）

---

## 4. AI 情绪分析输出协议

模型必须返回 JSON（禁止自由文本），建议结构：

```json
{
  "weights": {
    "joy": 0.12,
    "trust": 0.18,
    "fear": 0.10,
    "surprise": 0.08,
    "sadness": 0.14,
    "disgust": 0.07,
    "anger": 0.11,
    "anticipation": 0.20
  },
  "top3": ["anticipation", "trust", "sadness"],
  "confidence": 0.82,
  "rationale": [
    "...",
    "..."
  ]
}
```

后处理硬规则：

1. 必须包含 8 个键；缺失键补 0。
2. 对所有权重进行 clamp 到 [0,1]。
3. 归一化：sum(weights) = 1。
4. top3 由系统根据排序重算，不信任模型原值。
5. 置信度缺失时默认 0.6。

---

## 5. 玩家猜测规则

1. 每轮玩家必须选择 3 种情绪（不足 3 种不可确认）。
2. 猜测不会失败，不中断调酒流程。
3. 不设置猜错扣分（不降信任、不罚钱）。

命中定义：

- 令 guessed3 为玩家选择集合。
- 令 top3 为系统计算出的真实前三情绪集合。
- 命中数 h = |guessed3 ∩ top3|，h 取值 0~3。
- 令 surfaceSet 为当前可观察到的表象情绪集合（建议为 1~2 个）。
- 表象命中数 hs = |guessed3 ∩ surfaceSet|，hs 取值 0~3。

---

## 6. 调酒目标生成（由猜测驱动）

### 6.1 八情绪口感原型（初版）

每个情绪映射一个三维原型向量（thickness, sweetness, strength）：

- joy: (1, 2, 1)
- trust: (2, 1, 1)
- fear: (0, 0, 2)
- surprise: (1, 1, 2)
- sadness: (2, 0, 1)
- disgust: (1, 0, 2)
- anger: (0, 0, 3)
- anticipation: (1, 2, 2)

### 6.2 合成公式

玩家猜测 3 种情绪，默认等权合成目标向量：

T = (E1 + E2 + E3) / 3

再将目标向量转换为判定区间（建议每维容差 ±0.5）。

说明：

1. 后续可升级为非等权（玩家可指定主情绪权重）。
2. 若目标在当前材料池无解，执行“逐步放宽”策略（与现有 solvable target 机制一致）。

---

## 7. 奖励与信任规则

按照命中数 h 进行奖励分档：

- h=3：tipMultiplier = 1.80，trustGain = 0.18
- h=2：tipMultiplier = 1.40，trustGain = 0.10
- h=1：tipMultiplier = 1.15，trustGain = 0.05
- h=0：tipMultiplier = 1.00，trustGain = 0.02

表象命中额外信任（小额补贴，不影响小费倍率）：

- 每命中 1 个表象情绪，额外增加 extraTrustPerHit = 0.015
- 单杯额外信任上限 extraTrustCap = 0.03

即：

- extraTrust = min(0.015 × hs, 0.03)
- finalTrustGain = trustGain + extraTrust

结算方式：

1. 小费 = 基础小费 × tipMultiplier × 调酒质量系数。
2. 信任增加 = finalTrustGain（并可叠加调酒质量微调）。
3. 禁止因猜测错误产生负向惩罚。
4. 表象命中只提供小额正反馈，底层命中仍是主要收益来源。

---

## 8. 平局与边界处理

1. 权重并列时，使用固定优先级打破平局：
   joy > trust > anticipation > surprise > sadness > fear > anger > disgust
2. 若模型返回异常或解析失败：使用均匀分布（每项 0.125）。
3. 若玩家未满 3 选：确认按钮禁用。
4. 若角色资料极短：仍可分析，但 confidence 上限建议限制为 0.65。

---

## 9. 存档与数据结构建议

新增字段建议：

```json
{
  "emotionSchemaVersion": 2,
  "currentEmotionWeights": {
    "joy": 0.12,
    "trust": 0.18,
    "fear": 0.10,
    "surprise": 0.08,
    "sadness": 0.14,
    "disgust": 0.07,
    "anger": 0.11,
    "anticipation": 0.20
  },
  "currentEmotionTop3": ["anticipation", "trust", "sadness"],
  "lastPlayerGuess3": ["joy", "trust", "anticipation"],
  "lastGuessHitCount": 2
}
```

兼容策略：

1. 旧存档若无 emotionSchemaVersion，按 v1 读取并迁移到 v2。
2. 迁移失败时回退默认均匀权重，不阻断读取。

---

## 10. 验收标准（MVP）

功能验收：

1. 能随机读取角色 YAML 并稳定产出 8 维权重。
2. 玩家可稳定选择 3 情绪并进入调酒。
3. 奖励档位严格按 h=0~3 生效。
4. 表象命中额外信任按 hs 正确结算，并遵守单杯上限。
5. 无“猜错导致流程中断/扣罚”的情况。

数据验收：

1. 所有运行时情绪 ID 均在 8 种集合内。
2. 每轮权重归一化误差小于 1e-6。

体验验收：

1. 连续 20 局测试无阻断错误。
2. 玩家能感知“猜对越多，收益越高”的反馈差异。

---

## 11. 建议的实现顺序

1. 先完成角色 YAML 随机读取 + 情绪权重分析（独立脚本）。
2. 再接入游戏内猜测 3 情绪流程（无惩罚版）。
3. 最后替换调酒目标生成与奖励结算。
4. 收集 100 局自动测试数据后再调倍率。
